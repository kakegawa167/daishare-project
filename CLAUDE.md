# ダイシェア モバイルアプリ — Claude Code 作業ルール

## プロジェクト概要

台車の個人間レンタルマッチングサービス「ダイシェア」のモバイルアプリ。
詳細は `docs/design.md` を参照。

---

## 必須ルール

### 作業進行の原則

- **確認なしで作業を進める**。ユーザーの承認を待たず、判断できることはすべて自律的に実行する
- 後から修正が必要な場合はユーザーが指示するため、実装・ドキュメント更新・コミットまで一気に完了させる
- 「〜しますか？」「〜でよいですか？」といった確認は不要。ただし **破壊的操作**（本番DBの変更・force push等）は事前確認する

### タスク管理（docs/tasks.md）

- タスクに着手したら `[ ]` → `[→]` に変更する
- タスクが完了したら `[→]` → `[x]` に変更する
- **タスクの完了・着手・新規追加のたびに必ず `docs/tasks.md` を更新する（指示がなくても自動で行う）**
- 複数タスクをまとめて完了した場合も、すべて反映してから次に進む
- ページ末尾の「進捗サマリー」テーブルも合わせて更新する
- 新しい作業が発生したら、適切な Phase に新規タスクとして追加する

### 設計書管理（docs/design.md）

- アーキテクチャ・DB・API・画面構成などの設計を変更した場合は、**必ず `docs/design.md` を更新する（指示がなくても自動で行う）**
- バージョン番号（`> バージョン: X.X.X`）をインクリメントする（パッチ: バグ修正・軽微変更、マイナー: 機能追加・画面変更、メジャー: 大規模リファクタ）
- 設計変更の内容は変更箇所のみ更新し、無関係な箇所は触らない

### エラー記録（docs/error.md）

- エラーが発生・解決した場合は **必ず `docs/error.md` に記録する（指示がなくても自動で行う）**
- 記録する内容:
  - エラーID（ERR-NNN 連番）
  - 発生日時
  - 発生画面・ファイル
  - 症状
  - 原因
  - 対応方法
  - 注意点（再発防止・注意事項があれば）
  - 対象ファイル
- 同種のエラーが再発した場合は既存エントリを更新する（重複追加しない）

### 課題管理（docs/issues.md）

- Claude が自律的に解決できない課題・ブロッカーが発生した場合は **必ず `docs/issues.md` に記録する（指示がなくても自動で行う）**
- 記録する内容:
  - 課題ID（ISS-NNN 連番）
  - 発生日時
  - ステータス（`open` / `in-progress` / `resolved`）
  - 課題内容（何が問題か）
  - 影響範囲
  - 必要なアクション（ユーザーが行う必要があること）
  - 解決日時・解決方法（resolved になったら追記）
- 解決したら `resolved` に更新し、解決方法を追記する

### Git

- タスク完了・設計更新・エラー解決・課題解決のたびにコミットする
- コミットメッセージは英語で簡潔に記述する
- push はユーザーから指示があった場合のみ実行する

## ドキュメント更新タイミング一覧

| イベント                       | 更新対象              |
| ------------------------------ | --------------------- |
| 設計変更（API / DB / 画面）    | `docs/design.md`      |
| タスク着手・完了・新規追加     | `docs/tasks.md`       |
| エラー発生・解決               | `docs/error.md`       |
| ブロッカー・要対応課題の発生   | `docs/issues.md`      |
| 上記すべての後                 | git commit            |

## 技術スタック

- **モバイル**: React Native + Expo SDK 56 (TypeScript)
- **バックエンド**: FastAPI (Python 3.12) — Railway にホスティング
- **DB**: Supabase PostgreSQL（FastAPI が asyncpg で直接接続）
- **Auth / Realtime / Storage**: Supabase
- **コンテナ**: Docker + Docker Compose（ローカル開発）
- **マイグレーション**: Alembic
